1//! # KeOS: KAIST Educational Operating System
2//!
3//! Welcome to the **KeOS** project!
4//!
5//! Operating systems (OS) form the backbone of modern computing, managing
6//! hardware resources and providing a foundation for applications to run
7//! efficiently. Understanding how an OS works is crucial for grasping the inner
8//! workings of computer systems and software performance.
9//!
10//! **KeOS** is designed to offer a hands-on learning experience with core OS
11//! components. Through this project, you will explore the fundamental
12//! principles of OS design while building a minimal but functional operating
13//! system from the ground up.
14//!
15//! We prioritize simplicity, focusing on the most essential aspects of OS
16//! development. You won't have to worry about handling obscure edge cases or
17//! hidden test cases. **The score you receive after running the grading scripts
18//! is your final score.** Our goal is to make this project as straightforward
19//! as possible. If you have suggestions on how we can further minimize
20//! unnecessary complexity and focus on the core concepts, we encourage your
21//! feedback.
22//!
23//! ## Project Structure
24//!
25//! The KeOS project is divided into five projects:
26//!
27//! 1. **[`System Call`]** – Learn how the OS interacts with user applications.
28//! 2. **[`Memory Management`]** – Implement basic memory management and
29//!    user-space execution.
30//! 3. **[`Advanced Memory Management`]** – Expand the KeOS's memory management
31//!    system with advanced features.
32//! 4. **[`Process Management`]** – Implement the advanced process management,
33//!    including round robin scheduler and sychronization primitives.
34//! 5. **[`File System`]** – Develop a simple yet functional filesystem for data
35//!    storage.
36//!
37//! Each project builds upon the previous ones, helping you progressively
38//! develop a deeper understanding of OS design.
39//!
40//! ## Why Rust?
41//!
42//! We have chosen **Rust** for this project because of its **memory safety**,
43//! **zero-cost abstractions**, and most importantly, its **concurrency model**.
44//! These features make Rust an excellent choice for **operating system
45//! development**.
46//!
47//! In traditional system programming languages, concurrency and memory bugs
48//! such as data races, use-after-free errors, and null pointer dereferences are
49//! common. Rust prevents these issues at compile time by enforcing strict
50//! **ownership, borrowing, and lifetime rules**. This allows you to write
51//! **safe and efficient concurrent code** without sacrificing performance, and
52//! **reduces debugging time** for those bugs.
53//!
54//! By using Rust in **KeOS**, you will:
55//! - **Develop safe and efficient concurrent programs** without the risk of
56//!   data races.
57//! - **Avoid common concurrency pitfalls** such as race conditions.

166/// Enum representing errors that can occur during a kernel operation.
167///
168/// This enum is used to categorize errors encountered by the kernel operation.
169/// Each variant corresponds to a specific type of error that might
170/// occur during the handling of a kernel operation. These errors can be
171/// returned to the user program to indicate the nature of the failure.
172#[derive(Debug, Eq, PartialEq)]
173pub enum KernelError {
174    /// Operation is not permitted. (EPERM)
175    OperationNotPermitted,
176    /// No such file or directory. (ENOENT)
177    NoSuchEntry,
178    /// IO Error. (EIO)
179    IOError,
180    /// Exec format error. (ENOEXEC)
181    NoExec,
182    /// BAD file descriptor. (EBADF)
183    BadFileDescriptor,
184    /// Out of memory. (ENOMEM)
185    NoMemory,
186    /// Permission denied. (EACCES)
187    InvalidAccess,
188    /// Bad address. (EFAULT)
189    BadAddress,
190    /// Device or resource busy. (EBUSY)
191    Busy,
192    /// File exists. (EEXIST)
193    FileExist,
194    /// Not a directory. (ENOTDIR)
195    NotDirectory,
196    /// Is a directory. (EISDIR)
197    IsDirectory,
198    /// Invalid arguement. (EINVAL)
199    InvalidArgument,
200    /// Too many open files. (EMFILE)
201    TooManyOpenFile,
202    /// No space left on device. (ENOSPC)
203    NoSpace,
204    /// Broken pipe. (EPIPE)
205    BrokenPipe,
206    /// File name too long. (ENAMETOOLONG)
207    NameTooLong,
208    /// Invalid system call number. (ENOSYS)
209    NoSuchSyscall,
210    /// Directory not empty (ENOTEMPTY)
211    DirectoryNotEmpty,
212    /// File system is corrupted. (EFSCORRUPTED)
213    FilesystemCorrupted(&'static str),
214    /// Operation is not supported. (ENOTSUPP)
215    NotSupportedOperation,
216}
217
218impl KernelError {
219    /// Converts the [`KernelError`] enum into a corresponding `usize` error
220    /// code. The result is cast to `usize` for use as a return value in
221    /// system calls.
222    pub fn into_usize(self) -> usize {
223        (match self {
224            KernelError::OperationNotPermitted => -1isize,
225            KernelError::NoSuchEntry => -2,
226            KernelError::IOError => -5,
227            KernelError::NoExec => -8,
228            KernelError::BadFileDescriptor => -9,
229            KernelError::NoMemory => -12,
230            KernelError::InvalidAccess => -13,
231            KernelError::BadAddress => -14,
232            KernelError::Busy => -16,
233            KernelError::FileExist => -17,
234            KernelError::NotDirectory => -20,
235            KernelError::IsDirectory => -21,
236            KernelError::InvalidArgument => -22,
237            KernelError::TooManyOpenFile => -24,
238            KernelError::NoSpace => -28,
239            KernelError::BrokenPipe => -32,
240            KernelError::NameTooLong => -36,
241            KernelError::NoSuchSyscall => -38,
242            KernelError::DirectoryNotEmpty => -39,
243            KernelError::FilesystemCorrupted(_) => -117,
244            KernelError::NotSupportedOperation => -524,
245        }) as usize
246    }
247}

249/// The given `isize` does not indicate an [`KernelError`].
250#[derive(Debug, Eq, PartialEq)]
251pub struct TryFromError {
252    e: isize,
253}
254
255impl TryFrom<isize> for KernelError {
256    type Error = TryFromError;
257
258    // Required method
259    fn try_from(value: isize) -> Result<Self, Self::Error> {
260        match value {
261            -1 => Ok(Self::OperationNotPermitted),
262            -2 => Ok(Self::NoSuchEntry),
263            -5 => Ok(Self::IOError),
264            -8 => Ok(Self::NoExec),
265            -9 => Ok(Self::BadFileDescriptor),
266            -12 => Ok(Self::NoMemory),
267            -13 => Ok(Self::InvalidAccess),
268            -14 => Ok(Self::BadAddress),
269            -16 => Ok(Self::Busy),
270            -17 => Ok(Self::FileExist),
271            -20 => Ok(Self::NotDirectory),
272            -21 => Ok(Self::IsDirectory),
273            -22 => Ok(Self::InvalidArgument),
274            -24 => Ok(Self::TooManyOpenFile),
275            -28 => Ok(Self::NoSpace),
276            -32 => Ok(Self::BrokenPipe),
277            -36 => Ok(Self::NameTooLong),
278            -38 => Ok(Self::NoSuchSyscall),
279            -39 => Ok(Self::DirectoryNotEmpty),
280            -117 => Ok(Self::FilesystemCorrupted("")),
281            -524 => Ok(Self::NotSupportedOperation),
282            e => Err(TryFromError { e }),
283        }
284    }
285}

296/// A builder for system configuration settings.
297///
298/// The [`SystemConfigurationBuilder`] struct provides an interface for
299/// configuring various system-wide settings before initialization.
300/// This builder is typically used during system setup to configure fundamental
301/// aspects like scheduling policies, memory management, and other system-wide
302/// parameters.
303pub struct SystemConfigurationBuilder {
304    _p: (), // Prevents external instantiation while allowing future extensions.
305}
306
307impl SystemConfigurationBuilder {
308    /// Sets the system-wide scheduler.
309    ///
310    /// This function configures the default scheduler with a custom scheduler
311    /// implementation. It is expected that the provided scheduler
312    /// implements the [`Scheduler`] trait and has a `'static` lifetime,
313    /// meaning it must outlive all references.
314    pub fn set_scheduler(self, scheduler: impl Scheduler + 'static) {
315        unsafe {
316            thread::scheduler::set_scheduler(scheduler);
317        }
318    }
319}

321/// The entry of the KeOS for bootstrap processor.
322#[unsafe(no_mangle)]
323#[allow(clippy::missing_safety_doc)]
324pub unsafe fn rust_main(core_id: usize, regions: abyss::boot::Regions, cmd: Option<&'static [u8]>) {
325    info!(
326        "\x1bc\n\
327__  __     ___  ____  \n\
328| |/ /___ / _ \\/ ___| \n\
329| ' // _ \\ | | \\___ \\ \n\
330| . \\  __/ |_| |___) |\n\
331|_|\\_\\___|\\___/|____/ \n\
332\n\
333KAIST educational Operating System\n\
334Copyright 2025 Computer Architecture and Systems Lab\n"
335    );
336    // Init memory.
337    unsafe {
338        info!("Memory: init memory.");
339        crate::mm::init_mm(regions);
340        if let Some(cmd) = cmd {
341            KERNEL_CMDLINE = CString::from_vec_with_nul(cmd.into()).ok();
342        }
343    }
344    info!("Devices: init pci devices.");
345    unsafe {
346        abyss::dev::pci::init();
347    }
348    // Load debug symbols
349    info!("Panicking: Load debug symbols.");
350    if !crate::lang::panicking::load_debug_infos() {
351        warning!("Failed to read kernel image. Disabling stack backtrace.");
352    }
353
354    unsafe extern "Rust" {
355        fn main(conf_builder: SystemConfigurationBuilder);
356    }
357    unsafe {
358        main(SystemConfigurationBuilder { _p: () });
359        abyss::boot::ap_init();
360        // Kill the kernel low address.
361        (abyss::addressing::Pa::new({
362            unsafe extern "C" {
363                static mut boot_pml4e: u64;
364            }
365            boot_pml4e as usize
366        })
367        .unwrap()
368        .into_kva()
369        .into_usize() as *mut mm::page_table::Pml4e)
370            .as_mut()
371            .unwrap()
372            .set_flags(mm::page_table::Pml4eFlags::empty());
373        println!(
374            "Command line: {}",
375            KERNEL_CMDLINE
376                .as_ref()
377                .and_then(|cmd| cmd.to_str().ok())
378                .unwrap_or("")
379        );
380    }
381
382    crate::interrupt::register(32, |_| scheduler().timer_tick());
383    crate::interrupt::register(126, mm::tlb::handler);
384    crate::interrupt::register(127, |_regs| { /* no-op */ });
385    BOOT_DONE.store(true, core::sync::atomic::Ordering::SeqCst);
386    // Now kernel is ready to serve task.
387    crate::thread::scheduler::idle(core_id);
388}

403// Test utilities
404#[doc(hidden)]
405pub trait TestCase
406where
407    Self: Sync + Send,
408{
409    fn name(&'static self) -> &'static str;
410    fn run(&'static self, task: Box<dyn Task>) -> bool;
411}
412
413impl<T> TestCase for T
414where
415    T: Fn() + Send + Sync + 'static,
416{
417    fn name(&'static self) -> &'static str {
418        core::any::type_name::<T>()
419    }
420    fn run(&'static self, task: Box<dyn Task>) -> bool {
421        print!("test {} ... ", core::any::type_name::<T>());
422        if crate::thread::ThreadBuilder::new(core::any::type_name::<T>())
423            .attach_task(task)
424            .spawn(self)
425            .join()
426            == 0
427        {
428            println!("ok");
429            true
430        } else {
431            println!("FAILED");
432            false
433        }
434    }
435}

437/// A driver for running tests.
438pub struct TestDriver<T: Task + Default + 'static> {
439    _t: core::marker::PhantomData<T>,
440}
441
442impl<T: Task + Default + 'static> TestDriver<T> {
443    /// Run the given tests.
444    pub fn start<const TC: usize>(tests: [&'static dyn TestCase; TC]) {
445        crate::thread::ThreadBuilder::new("test_main").spawn(move || {
446            let tests = match unsafe { KERNEL_CMDLINE.as_ref() } {
447                Some(cmd) if cmd.to_str() != Ok("") => {
448                    let filter = cmd
449                        .to_str()
450                        .expect("Failed to parse cmd")
451                        .split(" ")
452                        .collect::<BTreeSet<_>>();
453                    tests
454                        .iter()
455                        .filter(|test| {
456                            let name = test.name();
457                            let r = name.split("::").next().map(|n| n.len() + 2).unwrap_or(0);
458                            filter.contains(&name[r..])
459                        })
460                        .collect::<Vec<_>>()
461                }
462                _ => tests.iter().collect::<Vec<_>>(),
463            };
464            let (total, mut succ) = (tests.len(), 0);
465            println!(
466                "Running {} test{}",
467                total,
468                if total == 1 { "" } else { "s" }
469            );
470
471            for test in tests {
472                if test.run(Box::new(T::default())) {
473                    succ += 1;
474                }
475            }
476            println!(
477                "test result: {}. {} passed; {} failed",
478                if total == succ { "ok" } else { "FAILED" },
479                succ,
480                total - succ
481            );
482
483            unsafe {
484                abyss::x86_64::power_control::power_off();
485            }
486        });
487    }
