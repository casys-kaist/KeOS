1//! Project 3: Hardware Virtualization.
2//!
3//! In this project, you will implement the hardware-based virtualization and
4//! run guest KeOS (gKeOS) on KeV hypervisor.
5//! The term "hardware-based virtualization" refers to the process of
6//! virtualizing platform hardware to run various operating systems on single
7//! hardware with hardware supports. Through this, users can run multiple
8//! operating systems on a single hardware platform as if they were running on
9//! multiple physical machines with minimal performance degradation.
10//!
11//! ## Background
12//! ### Hardware-based virtualization
13//! Hardware-based virtualization refers to the method of providing a virtual
14//! computing environment through the use of hardware, typically required for
15//! the operations of the virtual machines. This is accomplished using
16//! technologies such as Intel-VT or AMD-V, which provide the ability to run
17//! codes on another privilege level, enabling full virtualization without the
18//! need for OS modification such as binary translation or para-virtualization,
19//! with little or minimum performance degradations.
20//!
21//! Although hardware-based virtualization supports many hardware features,
22//! trap-and-emulate is still required as it is difficult to virtualize all
23//! instructions through the hardware. Therefore, the hardware assists in
24//! virtualization by focusing on frequently occurring or performance-critical
25//! instructions. One such example is page table modifications. In the past,
26//! shadow page table was used to emulate every memory access permission for
27//! each page table entry, but with the hardware-based virtualization, extended
28//! page table is used to reduce MMU overhead by allowing the hardware to
29//! perform these permission checks on behalf of the software.
30//!
31//! ## Getting started
32//! When you run the following command lines in the project3 directory, keos
33//! will panic with "not yet implemented" message. 
34//! 
35//! ```/bin/bash
36//! $ cargo run --target ../.cargo/x86_64-unknown-keos.json
37//! ```
