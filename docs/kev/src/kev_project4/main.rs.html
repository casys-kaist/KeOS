1//! Project 5: Final project
2//!
3//! You have learned the basic components of the virtulaization.
4//! In the final project, you will do an open-ended final project based on what
5//! you have learned. For the final project, you should add a significant piece
6//! of functionality to the kev, as described below. The project must (1) have a
7//! significant implementation component, specified by deliverables (2) concern
8//! operating systems and (3) be challenging and potentially research-worthy.
9//! You can modify any source code of the keos and kev infrastructure.
10//! For all projects, you must write a proposal that is accepted by the course
11//! staff.
12//!
13//! ## Candidates for final project
14//! Here is a list of possible tasks for a final project. Some are small enough
15//! that multiple options should be combined, especially if you have a larger
16//! group. You are most welcome to come up with other ideas. The actual project
17//! is up to you. Please pick something manageable. It is far better to complete
18//! your project to spec than it is to take on something too big and not have
19//! anything to show for it except excellent intentions (also true in the real
20//! world).
21//!
22//! - Port the kev to work on AMD SVM (the rough equivalent of Intel's VMX).
23//! - Implement support for an IOMMU to allow the guest to directly access
24//!   hardware.
25//! - Incorporate APICv support into kev.
26//! - Use a binary translator to support trap-and-emulate semantics on an x86
27//!   CPU without VMX or SVM support.
28//! - Implement network driver and work it on the kev.
29//! - Implement the nested virtualization to run kev on kev.
30//! - Support a smp in keos on kev.
31//!
32//! The work of students that do a particularly good project may be incorporated
33//! into future assignments (a good way to get your name on the credits page!).
34//! The project you choose must have a significant virtualization component. For
35//! example, you shouldn't simply port a user-level application that requires
36//! little or no kernel modification. You should email a proposal to the
37//! instructor by the notified deadline. The proposal must include: (1) The
38//! names of your group members; (2) What you want to do; and (3) What you are
39//! expecting to present (a list of deliverables). Please keep it short (no more
40//! than several paragraphs).
