1//! Cpuid vmexit controller.
2//!
3//! The cpuid instruction is a special instruction in x86 processors, used to
4//! obtain information about the processor and its features. When executed, the
5//! instruction returns data in the processor's registers, including the vendor,
6//! model number, and feature flags.
7//!
8//! The cpuid instruction takes one input argument (leaf) in the EAX register,
9//! which determines the type of information to retrieve. Different input values
10//! provide different information. For example, setting EAX to 0 returns a
11//! 12-character ASCII string representing the CPU manufacturer ID in EBX, EDX,
12//! and ECX registers, with "GenuineIntel" being the string for Intel
13//! processors.
14//!
15//! A guest operating system cannot execute the Cpuid instruction directly,
16//! instead causing a VM exit to the host, which must handle the instruction.
17//! This part aims to emulate the cpuid instruction in the guest by executing
18//! the instruction on the host side. However, not all the result could be
19//! forwarded. For example, when the host executes the Cpuid instruction with
20//! EAX = 1, the result contains the executing core's CPU ID, not the VCPU ID.
21//! In this case, the result needs to modified to contain the VCPU ID.
22//!
23//! ## Tasks
24//! Implement cpuid controller's handle method to emulate cpuid instruction.
25//! If the input to the instruction is EAX = 1, you must carefully handle the
26//! cpuid. Because it holds the cpu id of the current logical processor not
27//! virtual cpu id. It may be helpful to understand [how to obtain the CPU ID of
28//! the executing core.](/src/abyss/x86_64/intrinsics.rs.html) In addition, you
29//! **MUST** forward the vCPU instruction pointer (rip) to prevent it from
30//! executing the same instructions indefinitely.
