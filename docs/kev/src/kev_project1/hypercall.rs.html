1//! Hypercalls for project 2.
2//!
3//! Hypercall is a software trap from the guest operating system to hypervisor,
4//! similar to the syscall from the application to kernel. You can simply think
5//! hypercall as a "syscall" of the hypervisor and guest OS.
6//!
7//! In x86_64, guest OS can requests hypercall through the special instruction
8//! "vmcall". When guest OS executes "vmcall" instruction, it first vmexits to
9//! the hypervisor. After that, hypercall reads the registers and resolve the
10//! requested hypercall according to the its own abi for hypercall.
11//! And the hypervisor serves the requests and pass the control back to the
12//! guest OS through the "vmresume" instruction.
13//!
14//! Both the project now and the project afterwards, you will use the following
15//! abis for hypercall. %rax holds the hypercall number.
16//! %rdi, %rsi, %rdx, %r10, %r9, %r8 are the first and second arguments, and so
17//! on.
18//!
19//! ## Hypercall interface
20//! The core interface of hypercall is [`HypercallAbi`] and [`Hypercall`]
21//! traits. When the vcpu executes the vmcall, it traps into the vmexit handler
22//! of the host operating system. Then the vmexit control infrastructure of the
23//! kev forwards the given request to the [`Controller`] for the hypercall. When
24//! the [`Controller`] found that the given request is a hypercall, it probes
25//! the CPU state and resolve the information of the request through the
26//! [`Hypercall::resolve`]. After that the [`Controller`] passes the decoded
27//! hypercall request to the [`HypercallAbi::handle`]. The
28//! [`HypercallAbi::handle`] then finally handles the given requests.
29//!
30//! ## Tasks
31//! For this part, you are required to implement two hypercalls: the first halts
32//! the current vCPU, while the second prints a string to the console.
33//! The detailed Application Binary Interface (ABI) for each hypercall can be
34//! founded in the [`Hypercall`] code section. When you write to the console,
35//! you **MUST** proxy the console output through the [`PrinterProxy`].
36//! Otherwise, grading script may be failed.
37//!
38//! [`HypercallAbi`]: crate::vmexit::hypercall::HypercallAbi
39//! [`HypercallAbi::handle`]: crate::vmexit::hypercall::HypercallAbi::handle
40//! [`Hypercall`]: crate::vmexit::hypercall::Hypercall
41//! [`Hypercall::resolve`]: crate::vmexit::hypercall::Hypercall::resolve
42//! [`Controller`]: crate::vmexit::hypercall::Controller
43//! [`VmexitController`]: kev::vmexits::VmexitController
44//! [`ExitReason`]: kev::vmcs::ExitReason
45//! [`VmOps`]: kev::vm::VmOps

90    Print {
91        /// Buffer to print. Provides on rdi.
