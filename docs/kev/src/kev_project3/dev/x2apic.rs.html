1//! x2APIC Emulation
2//!
3//! ## Background
4//! ### Advanced Programmable Interrupt Controller
5//! In interrupt handling, the Advanced Programmable Interrupt Controller (APIC)
6//! represents a significantly improved alternative to the conventional
7//! Programmable Interrupt Controller (PIC). Originally, the 8235 PIC chip was
8//! responsible for interrupt processing, but due to speed and functionality
9//! considerations, interrupt handling routine was refined and integrated into
10//! the CPU in the cases of the local core interrupts.
11//!
12//! APIC is implemented as separate Local APIC and IOAPIC components.
13//! The former is embedded within each processor, while the latter is located in
14//! the system bus. APIC's primary objective is to process interrupts on
15//! processors and forward them to designated cores, as well as facilitate
16//! Inter-Processor Interrupt (IPI) communication in multicore environments.
17//!
18//! Over time, Intel has enhanced the APIC to newer versions such as xAPIC and
19//! x2APIC. One of the main differences between APIC and xAPIC is their
20//! functionality (Increased number of interrupt lines and flexibility in
21//! handling different types of interrupts), while x2APIC offers the ability to
22//! modify performance behavior through the use of MSR registers.
23//!
24//! #### LAPIC
25//! Each core has a dedicated Local APIC assigned to it, which can generate
26//! interrupts and trigger Inter-Processor Interrupts (IPIs) as well as handle
27//! up to 0-31 interrupt processing and 32-225 designated interrupt processing.
28//! Local APIC, which is directly connected to the processor's cores, performs
29//! interrupt-related tasks such as handling I/O devices, APIC Timer, IPI,
30//! Performance Monitoring Counter, and Thermal Sensor. LAPIC supports High
31//! Precision Event Timer (HPET), which is a high-precision timer that assists
32//! each core in obtaining accurate current time values without competing for
33//! one timer. If APIC is enabled and HPET is supported, the 8253 PIC is
34//! disabled.
35//!
36//! The definition of how each interrupt behaves can be configured via the Local
37//! Vector Table (LVT). LVT is controlled through memory access via MMIO for
38//! APIC, xAPIC, and similar devices, and via MSR registers for x2APIC.
39//! The Local Vector Table determines which hardware interrupt is forwarded to
40//! the core's interrupt pin.
41//!
42//! #### I/O APIC
43//! The I/O APIC is an Interrupt Controller that is responsible for delivering
44//! external device interrupts to the appropriate cores. Interrupts delivered
45//! through the I/O APIC are forwarded to the Local APIC based on the LVT
46//! configuration and ultimately delivered to the core. The I/O APIC is
47//! connected to the system bus and is responsible for delivering interrupts
48//! generated by hardware or I/O devices to the core.
49//!
50//! The I/O APIC has a Redirection Table that maps incoming interrupts from each
51//! device to a designated interrupt number. Unlike the 8253 interrupt
52//! controller, which supports only 16 external interrupts, the APIC supports up
53//! to 224 additional interrupts, excluding those assigned to hardware, through
54//! redirection.
55//!
56//! ### APIC Timer - TSC deadline mode
57//! APIC Timer modes are separated into three different modes for handling timer
58//! interrupts.
59//!
60//! #### Periodic Mode and One-shot Mode
61//! The Periodic Mode is a mode in which software (usually the operating system)
62//! requests the APIC to generate periodic timer interrupts by setting a
63//! specific time interval. For example, in the Periodic Mode, software requests
64//! the APIC to insert a timer interrupt every 1ms.
65//!
66//! The One-shot Mode is similar to the Periodic Mode but requests the timer
67//! interrupt to occur only once. Software requests the APIC to generate a timer
68//! interrupt after a certain time interval. For example, software requests the
69//! APIC to insert a timer interrupt 1ms later.
70//!
71//! #### TSC-Deadline Mode
72//! The TSC-Deadline Mode has a slightly different characteristic from the other
73//! two timer modes. Rather than sending a specific timer interval, this mode
74//! requests a timer interrupt when the value of the Time Stamp Counter (TSC) of
75//! the core exceeds a specific value. This method uses the TSC timer, which is
76//! more accurate than the other two methods that use the CPU frequency, and has
77//! a feature that is easy to avoid race conditions. For example, if the current
78//! TSC count is 100000, software requests the APIC to generate a timer
79//! interrupt when it becomes 100050, which is 100000 + Î±. Like one-shot mode,
80//! software should reprogram the next timer manually to re-trigger the next
81//! timer interrupt.
82//!
83//! Note that, the TSC deadline setting and initiate interrupts are separated.
84//! Initiate x2APIC timer interrupts are done by setting Model Specific Register
85//! (MSR register) with 0x832 to 0x10 (TSC-Deadline Mode). However, setting a
86//! deadline for Local APIC's TSC Deadline Mode is done by setting MSR with
87//! 0x6e0 to a timestamp for the next deadline.
88//!
89//! ## Tasks
90//! In this project, you are requested to implement timer interrupt
91//! virtualization for guest operating system. You need to implement timer
92//! interrupt virtualization for the guest by ensuring that a timer interrupt
93//! occurs every tick to schedule threads in the guest operating system.
94//!
95//! As the guest is unable to use the APIC timer in the host,
96//! the host must handle the initialization of the timer and setting of the
97//! deadline from the guest. If the TSC value exceeds the guest deadline, a
98//! virtual interrupt should be injected into the guest by a separate thread
99//! that runs in the hypervisor. This thread is spawned when the guest requests
100//! to set the timer as TSC deadline mode and tries to write the timer mode and
101//! interrupt vector into the MSR 0x832 (the MSR write is trapped to guest).
102//!
103//! After the thread is spawned,
104//! a deadline setting request via 0x6e0 MSR can be sent to the thread to inject
105//! interrupts when the TSC value is more than the set deadline. To share the
106//! deadline value between the created thread and the handler for 0x6e0 MSR, the
107//! [`channel`] API provided by KeV is used. Injection of the interrupt into the
108//! VM should only be done when the VM is not running, as the injected interrupt
109//! is handled when VmEntry occurs. To inject the timer interrupt into the
110//! running vCPU, the VMM must 1) [`kick`] the vCPU, 2) [`inject`] the
111//! interrupt, and then 3) [`resume`] the vCPU to execute the timer interrupt in
112//! the guest.
113//!
114//! [`channel`]: keos::thread::channel::channel
115//! [`kick`]: kev::vm::VmOps::kick_vcpu
116//! [`inject`]: kev::vcpu::VCpuOps::inject_interrupt
117//! [`resume`]: kev::vm::VmOps::resume_vcpu

142impl X2Apic {
143    pub fn attach(ctl: &mut msr::Controller) {
144        let inner = Arc::new(SpinLock::new(X2ApicInner {
145            apic_base_0x1b: APIC_BASE,
146            tx: None,
147        }));
148
149        // APIC_BASE MSR.
150        assert!(ctl.insert(
151            0x1B,
152            X2Apic {
153                inner: inner.clone()
154            }
155        ));
156        // TP.
157        assert!(ctl.insert(
158            0x808,
159            X2Apic {
160                inner: inner.clone()
161            }
162        ));
163        // eoi
164        assert!(ctl.insert(
165            0x80b,
166            X2Apic {
167                inner: inner.clone()
168            }
169        ));
170        // icr
171        assert!(ctl.insert(
172            0x830,
173            X2Apic {
174                inner: inner.clone()
175            }
176        ));
177        // timer
178        assert!(ctl.insert(
179            0x832,
180            X2Apic {
181                inner: inner.clone()
182            }
183        ));
184        // Susprious interrupt vector.
185        assert!(ctl.insert(
186            0x80F,
187            X2Apic {
188                inner: inner.clone()
189            }
190        ));
191        // lint0
192        assert!(ctl.insert(
193            0x835,
194            X2Apic {
195                inner: inner.clone()
196            }
197        ));
198        // lint1
199        assert!(ctl.insert(
200            0x836,
201            X2Apic {
202                inner: inner.clone()
203            }
204        ));
205
206        // tsc_deadline
207        assert!(ctl.insert(
208            0x6e0,
209            X2Apic {
210                inner: inner.clone()
211            }
212        ));
213    }
214}

216impl msr::Msr for X2Apic {
217    fn rdmsr(
218        &self,
219        index: u32,
220        _p: &dyn Probe,
221        _generic_vcpu_state: &mut GenericVCpuState,
222    ) -> Result<u64, VmError> {
223        let inner = self.inner.lock();
224        let r = match index {
225            0x1b => Ok(inner.apic_base_0x1b),
226            0x808 | 0x80b | 0x830 | 0x80f | 0x835 | 0x836 | 0x832 | 0x6e0 => Ok(0),
227            _ => todo!("{index:x}"),
228        };
229        inner.unlock();
230        r
231    }
232
233    fn wrmsr(
234        &mut self,
235        index: u32,
236        value: u64,
237        p: &dyn Probe,
238        generic_vcpu_state: &mut GenericVCpuState,
239    ) -> Result<(), VmError> {
240        let mut inner = self.inner.lock();
241        match index {
242            0x1b => inner.apic_base_0x1b = value,
243            // ICR
244            0x830 => {
245                let icr = ICR::from_bits_truncate(value as u32);
246                let (dst, _ipi) = ((value >> 32) as u32, value as u8);
247                match icr.mode() {
248                    ICRMode::Init => (),
249                    ICRMode::StartUp => {
250                        let entry = unsafe {
251                            (core::ptr::read_unaligned(
252                                p.gpa2hva(&generic_vcpu_state.vmcs, Gpa::new(0x467).unwrap())
253                                    .unwrap()
254                                    .into_usize() as *const u16,
255                            ) << 4)
256                                | (core::ptr::read_unaligned(
257                                    p.gpa2hva(&generic_vcpu_state.vmcs, Gpa::new(0x469).unwrap())
258                                        .unwrap()
259                                        .into_usize()
260                                        as *const u16,
261                                ) & 0xf)
262                        };
263                        let _ = generic_vcpu_state
264                            .vm
265                            .upgrade()
266                            .unwrap()
267                            .start_vcpu(dst as usize, entry);
268                    }
269                    e => unimplemented!("{e:?}"),
270                }
271            }
272            0x832 => {
273                let mode = (value >> 17) & 0b11;
274                let int = value as u8;
275                assert_eq!(mode, 0b10, "Only tsc deadline mode is supported.");
276                let (tx, rx) = channel(1);
277                todo!();
278
279                ThreadBuilder::new("timer_intr").spawn(move || {
280                    // Hint:
281                    //    - Receive the deadline from the rx.
282                    //    - Wait until time stamp exceeds the deadline.
283                    //    - You can get time stamp count with _rdtsc().
284                    //    - Kick vcpu and inject the interrupt #int to the vcpu.
285                    //    - Resume vcpu.
286                    todo!();
287                });
288                inner.tx = Some(tx);
289            }
290            0x6e0 => {
291                todo!()
292            }
293            0x808 | 0x80b | 0x80f | 0x835 | 0x836 => (),
294            _ => todo!("{index:x}"),

296        inner.unlock();

301bitflags::bitflags! {
302    struct ICR: u32 {
303        const DEST_1 = 1 << 19;
304        const DEST_0 = 1 << 18;
305        const LEVEL = 1 << 15;
306        const ASSERT = 1 << 14;
307        const PHYSICAL = 1 << 11;
308        const MODE_2 = 1 << 10;
309        const MODE_1 = 1 << 9;
310        const MODE_0 = 1 << 8;
311    }
312}

314impl ICR {
315    fn mode(&self) -> ICRMode {
316        match (self.bits() >> 8) & 7 {
317            0b000 => ICRMode::Fixed,
318            0b010 => ICRMode::Smi,
319            0b100 => ICRMode::Nmi,
320            0b101 => ICRMode::Init,
321            0b110 => ICRMode::StartUp,
322            _ => ICRMode::Reserved,
323        }
324    }
325}
