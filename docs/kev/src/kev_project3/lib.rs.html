1//! Project 4: Interrupt and I/O virtualization.
2//!
3//! In this project, you will implement timer interrupt virtualization and I/O
4//! virtualization for a simple block device.
5//!
6//! When KeOS is running as guest OS, all interrupts are trapped into the KeV
7//! hypervisor requiring that the interrupts are handled properly.
8//! In project3, internal interrupts generated by guest OS are ignored in the
9//! hypervisor for simplicity of the project. So there are no supports of
10//! multi-threading due to the absence of the timer interrupt. But now, it is
11//! the time to handle the parts to support scheduling in the guest.
12//!
13//! In addition to interrupt virtualization, I/O virtualization is needed to
14//! support higher performance device operations within guest operating system.
15//! Without I/O virtualization, All privileged instruction in an I/O operation
16//! are trap-and-emulated by hypervisor causing several context switching
17//! overhead. In the project4, We use para-virtualized I/O model to handle the
18//! I/O operations with a simple block device.
19//!
20//! ## Getting started
21//! When you run following command lines in the project4 directory, keos will be
22//! panic with "not yet implemented" message. 
23//! 
24//! ```/bin/bash
25//! $ cargo run --target ../.cargo/x86_64-unknown-keos.json
26//! ```
27//!
