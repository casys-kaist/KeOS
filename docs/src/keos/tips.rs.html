1//! Useful tips for implementing KeOS Projects.
2//!
3//! ## Implementation Strategy
4//!
5//! We recommend using a **"TODO-driven" approach** to implement project
6//! systematically. This method ensures an **incremental and structured**
7//! development process:
8//!
9//! 1. **Run the code** and identify `todo!()` placeholders that cause panics.
10//! 2. **Implement the missing functionality**, ensuring it aligns with the
11//!    expected behavior described in the project requirements.
12//! 3. **Repeat** steps 1 and 2 until all test cases pass and the system behaves
13//!    correctly.
14//!
15//! This approach allows you to build your project **one step at a time**,
16//! making debugging and understanding the system easier.
17//!
18//! ## Stopping an execution
19//!
20//! When KeOS got stuck in deadlock or does not automatically shut down after
21//! it panicked, you may need to forcibly shut down the QEMU.
22//!
23//! For execution in `cargo grade` or `cargo run` without argument in project 5,
24//! press **Ctrl-C** to stop execution.
25//!
26//! Otherwise, such as running KeOS by `cargo run` in project 1-4, press
27//! **Ctrl-A**, then press **X** to stop execution.
28//!
29//! ## Passing tests != bug-free correctness
30//!
31//! * The provided tests check basic functionality and semantics only. Passing
32//! them does not guarantee your code is bug-free.
33//! * Later components that build on your code may stress it in ways the tests
34//!   don’t cover, including edge cases.
35//! * This also applies across project milestones: a bug you see now may come
36//!   from a misimplementation in an earlier project—even if you received full
37//!   points at the time.
38//!
39//! ## Do not assign large variables on stack
40//!
41//! In **KeOS**, each process/thread is assigned a fixed execution stack of
42//! `STACK_SIZE` bytes. While KeOS attempts to detect stack overflows, its
43//! detection is not perfect. **A stack overflow may lead to mysterious kernel
44//! panics.** To avoid this:
45//! - **Avoid declaring large data structures on the stack.**
46//! ```rust
47//! let v: [u8; 0x200000]; // ERROR: This may cause a stack overflow
48//! ```
49//!
50//! - **Instead, allocate large data structures on the heap using `Box`.**
51//! ```rust
52//! let v = Box::new([0u8; 0x200000]); // OK: Allocates on the heap
53//! ```
